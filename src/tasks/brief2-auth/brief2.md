# привет
работа с авторизацией важная вещь, есть много способов ее реализовать
все способы тесно связаны с тем, как она устроена на бэкенде

но хорошо иметь опыт работы на фронте с основными видами 

```задача:
создать интерфейс с маршрутизацией на основе react-router-dom
при загрузке корневой странийы приложения, будет запрашиваться авторизация
пользователя, если пользователь еще не зарегистрирован - то также ему будет
предложено пройти процедуру регистрации

после успешной авторизации/регистрации - пользователю будет доступна
  страница с юзерами из первого задания (но также на ней может быть любой контент, главное, чтобы эта страница 
была доступна только авторизованным пользователям)

в случае если сессия авторизации истекла или если срок жизни токена истек 
и пользователь пробует зайти на защищенную страницу - его будет перебрасывать на 
форму авторизации


для авторизации можно использовать: собственное api (basic, jwt)
для формы регистрации - react-hook-form, final-form, formik
для маршрутизации - react-router-dom
для стейт-менеджмента (по желанию) - redux,redux-toolkit, mobx, recoil или jotai

в качестве ui библиотеки можно использовать ant-design, material ui
для стилизации scss, modules, styled-components, emotion 

задачи со звездочкой:
-в связке с бэком реализовать механизм ролей пользователя (юзер/админ/супер админ)
-в зависимости от прав авторизованного пользователя - раздать возможности удалять, редактировать
или добавлять пользователей (в бд юзеров из первого урока)

```
